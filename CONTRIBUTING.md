# Contributing to ChronoCam2

Thanks for your interest in contributing! This document explains how to propose changes.

## Ground Rules
- Be respectful and constructive in discussions.
- Keep changes focused and as small as possible.
- Add/update documentation for any user-facing change.
- Add or update tests when fixing bugs or adding features (if applicable).

## Getting Started
1. Fork the repo and create a feature branch.
2. Create a virtual environment and install dependencies.
3. Run the app and any tests to ensure things work locally.

## Commit Messages
- Use clear, imperative subject lines (e.g., "Add X", "Fix Y").
- Reference issues when applicable (e.g., "Fixes #123").

## Code Style
- Follow idiomatic Python.
- Prefer type hints for new or refactored code.
- Keep functions/components small and focused.

## Reporting Security Issues
- Do not open a public issue. See `SECURITY.md` for private disclosure instructions.
